.../pia/Config

[..] [files]

This directory contains severalkinds of configuration information:

  1. PIA `property' files.
  2. Site configurations, for various arrangements of the PIA server's ``website'' (URL hierarchy).
  3. Environment configurations, for various operating systems and environments.
  4. the ``Software Development Kit'', consisting of makefiles and other configuration files.

Annotated Contents

PIA Configuration Files

pia.props
A typical set of properties. Copy into your PIA user directory (~/.pia/Config on Unix) and edit to your liking.
pia.props.proxied
This is a property file designed for use in a PIA that is accessed through a proxy such as Apache.

Site Configuration Files

Site/
This subdirectory contains sample site configuration files, and include files used in site and subsite configurations.

Operating Systems & Environments

unix/
Configuration files for Unix or Linux users. Currently contains an Apache configuration file.

Software Development Kit

makefiles/
Templates to be included in Makefile's.

Conventions

In general, directories with capitalized or uppercase names belong to specific applications, while directories with lowercase names are more generic.
