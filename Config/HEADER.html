This directory contains three kinds of configuration information:

  1. Appliance configurations. Each appliance's configuration is in a directory with the name of the appliance's top-level agent.
  2. Environment configurations, for various operating systems and environments.
  3. the ``Software Development Kit'', consisting of makefiles and other configuration files.

Annotated Contents

PIA Configuration Files

pia.props
A typical set of properties. Copy into your PIA user directory (~/.pia/Config on Unix) and edit to your liking.
pia.props.proxied
This is a property file designed for use in a PIA that is accessed through a proxy such as Apache.

Appliances

Demos/
Configuration files for a PIA Demonstration appliance.
MB3/
Configuration files for a Media Board appliance.
CDRW/
Configuration files for a CD-RW or CD-R appliance.
Photo/
Configuration files for a ``photo-album'' appliance.
Printer/
Configuration files for a Web Printer.
WebWort/
Configuration files for a ``generic'' web appliance.

Operating Systems & Environments

Content and Makefile for various CD-ROM release formats.

Configuration files for Unix or Linux users. (This may eventually include multiple setups for ordinary user and developer.)
win95/, win31/, mac/ (eventually)

Software Development Kit

makefiles/
Templates to be included in Makefile's.
pia.props
Default properties (profile) file.
filemap.props
Default filemap (extension to MIME type) file.

Conventions

In general, directories with capitalized or uppercase names belong to specific appliances, while directories with lowercase names are more generic. In all cases a ``pia'' subdirectory contains the appliance-specific contents of the ``~pia'' directory, and ``pia/my'' the contents of the ``~pia/.pia'' directory--the names are changed to avoid having special characters in the names.

Subdirectories may also contain a setup script or Makefile.
