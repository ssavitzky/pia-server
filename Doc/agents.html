Guide to PIA Agents

This document describes the sample agents that ship with the PIA. PIA agents fall into three categories:

  • Standard agents that are automatically installed when the PIA is set up.
  • Optional agents that ship with the PIA and can be installed at the user's discretion.
  • Contributed agents made available by the PIA developer community.

Standard Agents

The agents currently available are:

  • ROOT
  • Admin
  • DOFS
    • Doc
    • PIA
  • Proxie
    • Toolbar
    • History
    • Cache

ROOT

The ROOT agent is the main agent: it serves pages from the root directory of the URL directory tree. It is also responsible for directing requests to the appropriate agents (any URL that doesn't start with the name of an agent must be for ROOT).

Like all agents, the ROOT agent has a data directory that appears as a ``virtual subdirectory'' called ``~/''. Since the ROOT agent's URL is ``/'', its data directory is addressed by the URL ``/~/''. The ROOT agent's data directory is the PIA's ``user-data directory,'' usually referred to as USER_DIR because that is the name of the environment variable that points to it. On Unix systems, the default user directory is .pia in your home directory.

Admin

The Admin agent is the ``server administration'' or ``control panel'' agent. It manages all the other agents that make up the PIA Agency.

DOFS

The DOFS (Document-Oriented File System) agent is the ``type'' (generic) agent for the specialized agents that serve local files. Requests starting with the name of a DOFS agent are served by appending the remainder of the URL to the ``root'' directory of the DOFS agent.

Doc

Doc is the most important DOFS agent, and is one of two that are started by default: its ``root'' directory is PIA_HOME/Doc, the PIA's top-level documentation directory.

PIA

PIA is the other standard DOFS agent; its root directory is PIA_HOME, the directory the PIA is installed in. On most Unix systems this will be a subdirectory of your home directory, especially if you are doing software development on the PIA. At sites where many users are using the PIA but not doing much development, this may be a shared directory. On a special-purpose ``appliance'', the PIA's files are typically in the home directory of a ``user'' with the login name ``pia''.

One may well ask why Doc and PIA are separate agents, since the documentation is also accessible as /PIA/Doc. The answer is that it may be desirable to limit access to the main PIA directory, or to move the documentation (which is bulky) to a directory -- or even a separate web server -- that all users can share.

The Proxie Agents

Proxie is the main agent in a group of related agents that operate when the PIA is used as a proxy for web browsing. The Proxie family of agents combine to make the PIA a powerful ``web browser's assistant,'' which was the original goal when the PIA was designed.

To use these agents, set your browsers to use your PIA (sometimes referred to as your Personal Information Agency when it's being used in this way) as your proxy. This setting is usually found under the Network options available under edit/preferences, options/network, or some similar menu. You need to specify both the host name and port--8888 by default-- where your agency is running.

Toolbar

The Toolbar agent is the ``type'' agent for agents that put a ``toolbar'' of links at the top of a page.

There are two Toolbar agents. The fileTools agent adds a toolbar to pages served from local files by DOFS agents in the PIA agency. The remoteTools agent adds a toolbar to remote pages proxied through your agency.

The remoteTools agent is also the agent responsible for displaying the URL of every HTML page you visit as the output of the pia command.

History Agent

The History agent tracks all web pages that you visit from any browser, allowing you to search those URLs by date.

Once you set your Information Agency as your proxy, browse to a few external web pages, then visit the History agent's home page. You should see a link to today's date (e.g. [Hist: / 1998 / 08 / 04]) The last component, the day, is a list of links to all the pages you visited on that day. The month and year are directories corresponding to that month and year.

Cache Agent

(The Cache agent may not be functional in versions of the PIA prior to release 2.1; it's still experimental.)

Unlike the History agent, which saves only URL's, the Cache agent saves entire pages. This can take up a lot of space, but it can speed things up if you go back to a page you've visited recently. Eventually we want to be able to display the differences if a page has changed since the last time you visited it.

Optional Agents

The PIA provides a simple form for installing agents. Look for it in Admin/install_agent. The optional agents currently shipping with the PIA include:

  • Swifty
  • HelloWorld
  • View
  • Demo

Swifty

The Swifty agent lets you easily customize your PIA home page. It offers the option of adding useful links, provides a custom page title and a decorative image.

HelloWorld

The HelloWorld agent lets you choose a custom greeting. This agent provides a simple example of form use for PIA developers.

View

The View agent lets you view documents after those document are processed with a tagset or script. Three of the ``views'' provided are as raw text, as a ``slide'' presentation, and as the documentation for a ``tagset''.

Demo

This agent serves as both a demonstration and testbed for the PIA's extended HTML markup language and its formatter/interpreter. Its home page provides links to pages that show examples of tag usage.

Contributed Agents

Agents contributed by the PIA developer community can be found in the directory Contrib/<host.name>/Agents Install these agents much as you would any of the optional agents provided by Ricoh.

Users can place a copy of their agent directories in Contrib/HOST.NAME/ and submit them to the PIA group for inclusion in the general release. Before attempting to install a contributed agent, copy its directory and files to PIA_ROOT/Agents or PIA_HOME/Agents.
