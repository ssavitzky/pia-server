Installing the PIA

This manual describes how to install the Platform for Information Applications (PIA). Developers intending to create applications should also consult the Author's Guide.

Topics covered include:

  • Before You Begin
  • Obtaining the Files
  • Installing the Files
  • Running the PIA
  • Questions, Problems and Comments

For information on use of the PIA, see the PIA Document List.

Before You Begin

You need the following software to run the PIA:

  • Java (A Java Runtime Environment, JRE 1.1.3 or higher, is needed to use the software. Programmers wishing to modify the code will need a Java Development Kit JDK.) Java for Solaris and Windows can be obtained from java.sun.com (direct links to JRE or JDK). Java for Linux can be obtained from www.blackdown.org
  • An operating system that supports sockets and long filenames. We have run the PIA software on Linux (Red Hat 4.2 -6.0 with the blackdown.org JDK), Windows 9X, and Solaris on SPARC. It does not seem to work on the Macintosh because of differences in file handling.
  • Utilities for unpacking the files (tar and gunzip on unix, winzip on Windows).

Obtaining the Files

The PIA files are available via HTTP, from http://www.risource.org/PIA/.

The latest stable releases are maintained in compressed form in the directories ftp://ftp.risource.org/PIA/pia_src.tgz. This is always a link to the most recent version.

Uncompressing the Files

Before the PIA files can be extracted from the tar file, it must first be uncompressed. For example:

gunzip pia_src.tgz

Alternative: CVS

If you are planning to do PIA development, it is better to obtain the files using the CVS version-control system. More information on this can be obtained from www.risource.org/PIA/cvs.shtml

The CVS repository is updated more frequently than the tar archive -- essentially whenever a developer feels that the portion of the PIA they have been working on is stable enough for others to try.

Installing the Files

The files are supplied in the form of a gzip'ed tar file. Set your working directory to the intended parent directory for the PIA files. Extract the files using tar. For example:

tar xvf pia_src

This creates the directory PIA/ under your current directory.

Directory Structure (inside the PIA directory)
Agents Subdirectories for standard agents and applications. Each subdirectory contains the collection of XML files (active pages) that work together to form an application.
Config Configuration information, including subdirectories for Makefiles and for XML site configuration files.
Contrib Used for contributed agents and other files that are not a part of the standard release.
bin Programs that actually run the PIA software either as a web server ( pia, pia.bat or as a command line filter process. Note that pia and process run under Unix only.)
lib Currently not being used.
src src/java/ contains the Java source files used in the PIA software. The main code is in src/java/org/risource/* corresponding to org.risource.* package names.
src/org and src/misc contain code referenced by the PIA software and included for convenience. src/app contains utility applications.

Setting Up

The PIA uses two environment variables:

  • PIA_HOME indicates the location of the PIA directory just installed.
  • PIA_ROOT indicates the location to which the PIA writes its data files.

The scripts in the bin directory that run the software attempt to set the PIA_HOME variable based on the location of the script or the current working directory. You may have to explicitly set this variable to the location of the PIA directory you just installed (e.g. PIA_HOME=~/PIA).

PIA_ROOT is automatically created by the PIA. You are free to reset its location if desired. On Unix, the default directory for PIA_ROOT is ~/.pia. On Windows, the default directory is c:\PIAUSERS\username (if a username is not available nobody is used).

If you use the -root command line option to specify PIA_ROOT, that setting takes precedence over the one you specify in the environment variable.

When the PIA starts, it looks for the pia.props under ~/.pia/Config. You can create a customizable copy in your home directory under ~/.pia/Config by copying pia/Config/pia.props and modifying it using a text editor. (Note that this example assumes PIA_ROOT is ~/.pia)

The default settings of the PIA are contained in PIA/Config/pia.props. These settings can be overridden by a local copy of pia.props in PIA_ROOT/Config/pia.props, by environment variables or on the command line. Command lines options can be given in the form of switches (e.g. -p 8000) or at the end of the command line in the form of prop=value (e.g. pia.port=8000.)

It is convenient to put the directory $PIA_HOME into your shell's search path. On Linux you can do this by putting the following into your ~/.bash_profile file: 

  # The following line will change depending on where the PIA is installed
  export PIA_HOME=$HOME/PIA
  export PATH=$PIA_HOME/bin:$PATH

You may also need to set up Java's CLASSPATH environment variable. The PIA's wrapper script will try to set one up for you, but whether it succeeds will depend on how clever your java implementation is. You don't have to have $PIA_HOME/src/java in your CLASSPATH; the wrapper script is smart enough to add it for you.

Running the PIA

On UNIX or Linux, once you have set up the PIA_HOME and PIA_ROOT environment variables you can launch the PIA with the command: ``$PIA_HOME/bin/pia'', or just ``pia'' if $PIA_HOME/bin is in your search path. A complete list of command-line options can be found by running $PIA_HOME/bin/pia -help at the shell prompt.

Under Windows 9x and NT, launch the PIA in a DOS window with the command: PIA_HOME/bin/pia.bat -- this is a batch file that makes use of the PIA_HOME and PIA_ROOT environment variables, and is similar in function to pia_wrapper on Unix.

The process command allows the PIA software to be used as a unix filter for processing HTML and XML documents. Under Windows 9x and NT the CLASSPATH must be set to PIA_HOME/src/java in order for process to run.

Using the PIA

The PIA acts as a Web server. Assuming you have gotten the PIA to run in its default configuration, you should now be able to browse to http://localhost:8888/ in order to access the PIA's own web pages. If the default configuration is different and you have run the PIA from a command line, it will have printed out a line like: 

  Point your browser to <URL: http://blackroot.crc.ricoh.com:8001>

after initializing, and you can cut-and-paste the URL on that line into your browser and go from there.

Some of the functions work best if you set your browser's proxies to the PIA as well. In most cases, http://localhost:8888 will work.
