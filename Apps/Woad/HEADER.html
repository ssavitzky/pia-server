Web-Organized Application Development. WOAD is a web-based ``Integrated Development Environment'' optimized for (but not limited to) web applications. As the name suggests, WOAD allows you to use the web to organize a development project, including its developer and user communities.

Annotated Contents

Directories

Icon/
contains icons and images used in the WOAD system.
Tools/
contains the tagsets and active web pages (forms) that implement listing, annotation, and editing functions in WOAD.
Words/
Aliased as /.words, this directory contains the index files that ship standardly with WOAD.

Tagsets

woad-xhtml.ts
WOAD's extensions to the PIA's XHTML. This tagset is used directly for .xh pages in the WOAD system, and is inherited by all other WOAD tagsets.
woad-index.xh
The tagset that implements .wi index files. It includes a wrapper element, because index files don't have a root element (in order to make them easy to append to).
woad-web.xh
The tagset that implements .ww (woad-web) annotation files.

Documents

Documents fall into three broad categories: .html files that are meant to be useable before WOAD is installed, the .xh files that implement the bulk of WOAD's functionality, and a limited number of ``home pages'' (also .xh files) that represent this directory under various configurations. We will document the latter here; the others are pretty obvious.

index.xh
The index page for this directory when viewed inside the PIA, i.e. when accessed as /Apps/Woad
woad.xh
The index page for this directory when viewed as /.Woad either ``mounted'' on a PIA-based application, or as a stand-alone woad server.
warrior.xh
The index page for this directory when viewed as the root of a Woad Warrior server. (currently incomplete).
