This directory contains the source code for classes and applications written in Java (which makes no firm distinction between a library class and the top-level class of an application).

Note:
This is the root of the Java package tree; it is not the java package. Your $CLASSPATH should contain this directory.

Annotated Contents

Directories

crc/
Classes developed at CRC.
misc/
Classes developed elsewhere.
org/
Classes developed by organizations that have chosen to abide by Sun's package naming convention (e.g. org.w3c...). Eventually we may join them.
makefiles/
Makefiles
(w3c)
An older version of the DOM. Eventually this will be removed.

Log Files

doc.log
Log from doing ``make doc''. Useful for tracking down problems with Javadoc comments.
holes.log
Log of all files that contain code or comments indicating that something is missing. Generated by ../app/tools/hole-report
lines.log
Line counts and filenames. Can be fed to xdu for a graphical display.
make.log
Log, preferably from doing ``make all JAVAFLAGS=-deprecation''
summary.log
Counts of holes and .java files; other summary information.
All log files can be regenerated with the command 
  make logs

Naming Conventions

Every effort has been made to reduce directory depth in order to accomodate Microsoft and ISO9660 restrictions. In particular, the hierarchy extends only three more levels below this directory (e.g. crc/dps/handle). There is room for one more, so that when/if we achieve `.org' status we can add that level to the hierarchy.
